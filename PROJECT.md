# 네잎클로버 찾기 게임 — 프로젝트 문서

> 모바일 힐링 게임 프로젝트 컨텍스트 & 개발 계획
> Last updated: 2026-05-16

---

## 1. 프로젝트 개요

### 컨셉
손그림 풍의 들판에서 세잎클로버 사이에 숨어 있는 네잎클로버를 찾는 모바일 힐링 게임.

### 비전
*네코 아츠메(고양이 모아모아)* 같은 차분한 컬렉션형 모바일 게임의 감성을, **클로버 찾기**라는 정적이고 명상적인 메커닉으로 구현. 짧은 세션에서 강한 **발견의 만족감**을 주는 것이 목표.

### 타겟 유저
- 일과 중 짧은 휴식이 필요한 사람
- 강한 도전이나 경쟁 없이 차분한 게임을 즐기는 유저
- *모은다는 행위* 자체에서 즐거움을 얻는 컬렉터형 유저

## 2. 게임 디자인

### 2.1 코어 루프
```
들판 진입 → 세잎/네잎 클로버 관찰 → 네잎 발견 → 수집/피드백 → 새 들판
```

### 2.2 핵심 메커닉
- **배치**: 한 화면 또는 스크롤 가능한 영역에 세잎클로버 다수 + 네잎클로버 1개 이상
- **상호작용**: 탭으로 클로버 선택
- **피드백**:
  - 정답(네잎): 시각적 강조 + 햅틱 + 사운드 + 컬렉션 등록
  - 오답(세잎): 가벼운 부정 피드백 (좌절감 최소화)
- **랜덤성**: 매 들판마다 클로버 위치 / 회전 / 색감 / 크기 변주

### 2.3 결정 사항
- ✅ 손그림 감성 아트 디렉션
- ✅ 별도 레벨/스테이지 분할 없음 — 끊김 없는 들판 경험
- ✅ **씬(scene)** 으로 환경 다양화: 풀밭, 산, 꽃밭 등
- ✅ Phase 1은 **항상 네잎클로버 1개 보장** (좌절감 없는 코어 검증)
- ✅ 컬렉션 시스템 도입 (씬별 / 일자별 / 이벤트별)
- ✅ 이벤트 예시: *크리스마스 — 삼색(빨강/초록/흰색) 세잎클로버 사이의 삼색 네잎클로버*

### 2.4 미결정 사항
| 항목 | 내용 | 결정 시점 |
|------|------|-----------|
| 세션 길이 | 30초 / 1분 / 5분? | Phase 1 플레이테스트 후 |
| 0개 모드 | 네잎 없는 들판 허용? | Phase 3 (난이도 모드 도입 시) |
| 컬렉션 분류 축 | 씬별/일자별/이벤트별 — 구체적 기준 | Phase 3 진입 전 |
| BM 구체화 | 네잎 위치 힌트 외 추가 수익화? | Phase 4 |
| 사운드 디렉션 | 자연음 / 미니멀 / 배경음악? | Phase 2 |

## 3. 시각 & 콘텐츠

### 3.1 아트 방향
- 손그림 / 수채화 / 잉크 텍스처 감성
- 참고: 네코 아츠메, Alto's Odyssey 등의 차분한 색감과 텍스처
- **Phase 1**: SVG 도형 기반의 기하학적 클로버로 메커닉 검증
- **Phase 2**: 손그림 에셋으로 교체

### 3.2 씬 시스템
환경별 분위기 차등:
- **풀밭 (기본)**: 초록 톤, 평화로움
- **산**: 흙/돌/이끼 톤, 차분함
- **꽃밭**: 컬러풀, 화사함
- *추후 추가 가능*: 강가, 숲속, 밤의 들판 등

각 씬은 클로버 색감 / 배경 텍스처 / 사운드를 차등 적용.

### 3.3 컬렉션 시스템
- 발견한 네잎클로버를 갤러리에 보관
- 분류 축 (잠정):
  - **씬별**: 어디서 발견했는지
  - **이벤트별**: 시즌/한정 이벤트
  - **일자별**: 발견 시점 기록
- 카테고리마다 시각적으로 차등화된 클로버 디자인

## 4. 기술 스택

### 4.1 선택: Phaser.js + Capacitor
**근거**:
- 웹 프론트엔드 배경 활용 → 학습 곡선 최소화
- Phaser는 2D 게임에 최적화 (스프라이트, 입력, 씬 관리 내장)
- Capacitor로 HTML5 → iOS/Android 네이티브 패키징
- 프로토타입을 **브라우저에서 바로 테스트** 가능

**대안 검토**:
- React Native + Skia: 가능하지만 게임 로직 작성 부담
- Flutter: 학습 곡선
- Unity: 과한 도구

### 4.2 주요 의존성 (예상)
- `phaser` — 게임 엔진
- `@capacitor/core`, `@capacitor/ios`, `@capacitor/android` — 네이티브 패키징
- `@capacitor/haptics` — 햅틱 피드백
- `vite` — 빌드 도구
- `typescript` — 타입 안정성

### 4.3 폴더 구조 (제안)
```
clover-game/
├── src/
│   ├── scenes/                 # Phaser 씬
│   │   ├── BootScene.ts
│   │   ├── FieldScene.ts       # 메인 들판
│   │   └── CollectionScene.ts  # 컬렉션 갤러리
│   ├── entities/               # 게임 오브젝트
│   │   ├── Clover.ts
│   │   └── CloverField.ts
│   ├── systems/                # 게임 로직
│   │   ├── FieldGenerator.ts   # 들판 절차적 생성
│   │   └── CollectionStore.ts  # 컬렉션 영속성
│   ├── assets/                 # 이미지/사운드 (Phase 2~)
│   ├── config/                 # 게임 설정값
│   └── main.ts
├── public/
├── capacitor.config.ts
├── package.json
└── PROJECT.md                  # 이 문서
```

## 5. 개발 로드맵

### Phase 1: 코어 루프 검증 (1~2주)
**목표**: *"찾는 행위 자체가 재밌는가?"* 에 답하기

**산출물**:
- 한 화면, 세잎클로버 50~100개 + 네잎클로버 1개
- 탭 감지 및 정/오답 피드백
- 새 들판 생성 버튼
- 브라우저에서 플레이 가능

**아트**: SVG 기하학 도형 (타원 3~4개 회전 배치). 손그림 ❌

**검증 방법**: 5~10명에게 플레이시키고 관찰

### Phase 2: 감각 완성도 (2~3주)
**목표**: *"만질 맛이 나는가?"*

**산출물**:
- 손그림 에셋 (세잎/네잎 클로버, 배경 텍스처)
- 햅틱 + 사운드 피드백
- 카메라 / 자연스러운 들판 스크롤
- 발견 시 만족스러운 애니메이션

### Phase 3: 컬렉션 & 진행감 (3~4주)
**목표**: *"다시 돌아오고 싶은가?"*

**산출물**:
- 컬렉션 갤러리 화면
- 씬 전환 (풀밭 / 산 / 꽃밭)
- 일일 발견 기록
- 로컬 영속성 (Capacitor Preferences 또는 SQLite)

### Phase 4: 출시 준비 & BM (4주~)
**산출물**:
- 네잎클로버 위치 힌트 (IAP 또는 광고 보상)
- 시즌/이벤트 시스템 (크리스마스, 봄 등)
- 푸시 알림
- iOS/Android 빌드 & 스토어 등록

## 6. Phase 1 첫 스프린트 — 구체 작업 단위

Claude Code로 바로 착수 가능한 단위:

### 6.1 프로젝트 부트스트랩
- Vite + TypeScript + Phaser 프로젝트 생성
- Capacitor 설정 (당장 네이티브 빌드는 안 해도 됨)
- 기본 폴더 구조 잡기

### 6.2 `Clover` 엔티티
- 작은 타원 3개(세잎) 또는 4개(네잎)를 중심점 기준으로 회전 배치하는 SVG 생성 함수
- 파라미터: `leaves: 3 | 4`, `hue`, `scale`, `rotation`
- Phaser `Container` 또는 동적 `Texture`로 변환

### 6.3 `FieldGenerator`
- N개의 클로버를 화면에 배치 (간단한 푸아송 디스크 샘플링 또는 그리드 지터)
- 정확히 1개의 네잎 보장
- 시드 기반 재생성 가능 (디버깅용)

### 6.4 `FieldScene`
- 위 생성기로 들판 렌더링
- 클로버 탭 입력 처리
- 정답: 강조 효과 + "찾았다!" 토스트 + 0.5초 후 다음 들판
- 오답: 작은 흔들림 또는 무반응
- "새 들판" 버튼 (디버깅용)

### 6.5 카운터
- 이번 세션에서 발견한 네잎클로버 수 표시 (영속성은 Phase 3에서)

**완료 기준**: 위 5개를 마치면 코어 루프 검증 가능. 친구들에게 보낼 수 있는 데모.

## 7. Claude Code 협업 노트

- **TypeScript 우선**: 게임 오브젝트의 타입 안정성 중요
- **Phase 1에서는 과도한 추상화 금지**: 빠르게 만들고 빠르게 검증
- **상태 관리**: Phase 1~2는 Phaser 내장 상태로 충분. Phase 3부터 Capacitor Preferences/SQLite 고려
- **아트 에셋**: Phase 1은 코드 생성, Phase 2부터 외부 에셋(`assets/`)
- **테스트**: Phase 1은 `FieldGenerator` 같은 순수 로직만 단위 테스트
- **커밋 단위**: 위 6.1~6.5의 각 작업이 PR/커밋 단위로 적절
