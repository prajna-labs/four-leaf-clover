> 외부 피드백(테스터/유저)과 본인 아이디어 중 *지금 결정 안 하는 모든 것*을 모음.
> 모두 반영하진 않음 — 의미 있는 것만 PROJECT.md / DECISIONS.md / commit으로
> graduate하고, 채택 안 한 것은 이유와 함께 남김.
>
> **상태**: 🆕 신규 / ✅ 반영 / ⏸ 보류 / ❌ 반려
> **시간 역순** — 최신이 위.
>
> **형식**:
> - 세션 헤더로 관련 항목을 묶음 (예: `## 2026-05-17 — Phase 1 1차 플레이테스트`).
>   환경/맥락(테스터 수, 기기, 배포 URL 등)을 헤더 본문에 적음.
> - 본인 메모도 날짜별 세션 헤더 밑에 모음.
> - 각 항목: `- {상태} [짧은 요약] — 출처: {source}. {원문/paraphrase}. (선택) 메모/맥락.`
> - ✅로 graduate한 항목은 링크 명시 (`→ DECISIONS.md '해시'`, `→ 커밋 '해시'` 등).

## 2026-05-19 — Phase 1 플레이테스트 피드백 (1차)

레퍼런스 이미지(유화풍 들판, 클로버 패턴 일러스트, 클로버 밀집 일러스트, 강아지+클로버 일러스트) 공유와 함께 수집된 피드백.

- 🆕 [플레이 타이머 표시] — 출처: 유저. 찾는 데 얼마나 걸렸는지 플레이 시간을 화면에 표시해달라는 요청.
- 🆕 [PC/모바일 밀도 불균형] — 출처: 유저. PC에서는 클로버가 너무 퍼져 있고, 모바일에서는 너무 빽빽하다. 뷰포트별 density 조정 필요.
- 🆕 [아트 방향 — 레퍼런스 이미지 수준] — 출처: 유저. 클로버·배경이 공유된 레퍼런스 이미지(유화풍·일러스트 스타일) 수준이면 좋겠다. Phase 1은 코드 생성 SVG로 가지만, Phase 2 아트 에셋 방향의 구체적인 레퍼런스로 확보.
- 🆕 [클로버 드래그 물리 연출] — 출처: 유저. 터치/스와이프로 클로버를 드래그하면 옆으로 밀리며 뒤에 가려진 것이 드러나는 모션. 현재 Phase 1은 단순 탭 판정이지만, 터치 상호작용의 핵심 재미 요소가 될 수 있음. Phase 2 후보.
- 🆕 [밀도 기반 난이도 구분] — 출처: 유저. 클로버 빽빽한 정도를 난이도 레벨로 활용하는 아이디어. PROJECT.md §2에서 아직 미결인 난이도 설계와 연결 가능.
- 🆕 [PC 맵 스크롤 탐색] — 출처: 유저. PC에서 전체 맵을 한 화면에 다 담지 않고, 화면 가장자리로 이동 시 맵의 다른 부분으로 스크롤되는 방식. PC 전용 UX 아이디어.
- 🆕 [무한 모드 — 넓은 맵 + 클로버 계속 생성] — 출처: 본인 아이디어. 난이도별 유한 맵을 기본으로 하되, 병렬로 무한 모드도 설계. 넓은 맵 위에서 클로버가 계속 생성되는 방식. 유한 모드(완료 만족감)와 무한 모드(흘러가는 힐링감) 두 가지 플레이 패턴을 동시에 지원할 수 있음.
- 🆕 [모바일 맵 탐색 불편 우려] — 출처: 유저. 모바일에서 맵의 다른 부분으로 이동하는 방식은 좁은 화면+터치 조작이라 불편할 것 같다는 우려. 위 PC 스크롤 아이디어는 모바일에 그대로 적용하면 안 된다는 제약 조건.
